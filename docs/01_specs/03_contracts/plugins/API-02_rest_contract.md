# API-02 REST 契約（プラグイン用・将来）

## 1. ドキュメント情報

| 項目 | 内容 |
|------|------|
| **ID** | API-02 |
| **関連ドキュメント** | API-01（枠組み）、ARC-02（プラグイン構成） |

本ドキュメントは**プラグイン用 API 契約**を扱う。**将来**プラグインがサーバー API を持つ場合に使用する。MVP ではエンドポイントは存在せず、「将来用」として 1 文書を用意する。

---

## 2. スコープ（MVP）

- MVP ではプラグインの判定は**ブラウザ内（Web Worker + Pyodide）で完結**するため、プラグイン専用のサーバーエンドポイントは不要である。
- 枠組みの API-01（解答送信は POST submit で枠組みがプラグインの判定アダプタを呼び出す）で足りる。

---

## 3. 将来の拡張方針

プラグインがサーバー側で処理を行う API を追加する場合の方針を以下に示す。

- **エンドポイント追加先**: **openapi_plugins.yaml** を 03_contracts/plugins/ に新設するか、既存の **openapi.yaml**（03_contracts 直下）にパスを追加する。プロジェクトでどちらを採用するかは、エンドポイント数や責任範囲に応じて決める。
- **API-02 の役割**: プラグイン用 API の説明・一覧を本ドキュメントに記載し、正本は上記 OpenAPI ファイルで管理する。
