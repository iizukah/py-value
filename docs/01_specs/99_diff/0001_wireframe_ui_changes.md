# 0001 ワイヤーフレーム UI 変更（仕様書との差分）

## 日付

2026-02-07

## 概要

ワイヤーフレーム（docs/.drafts/wireframes.html）に以下の UI を追加・変更した。現時点では仕様書（04_ui_ux）には未反映。**後で仕様書を修正する想定**で、本 diff に記録する。

---

## 1. 問題一覧（SC-001 / CP-003, CP-004）

| 項目 | 現行仕様書 | ワイヤーフレームでの変更 |
|------|------------|--------------------------|
| レイアウト | 問題カードの縦並び（一覧） | **タイル配置で flex**。横に複数タイルが並ぶグリッド。 |
| 問題文 | 仕様では「タイトル・難易度・バッジ・リンク」 | **問題文の冒頭**（抜粋）をカードに表示する。「問題文の冒頭...」のように省略表示。 |
| バッジ位置 | カード内に表示 | **バッジをタイルの中央**に配置する。 |

**反映先（予定）**: UX-01 SC-001 の「主要 UI 要素」、UX-02 CP-003・CP-004 の説明。

---

## 2. 正解画面・完了画面のレイアウト（SC-003, SC-004）

| 項目 | 現行仕様書 | ワイヤーフレームでの変更 |
|------|------------|--------------------------|
| 配置 | 特になし | **正解・バッジ表示（SC-003）と完了画面（SC-004）のブロックを画面中央**にレイアウトする。 |

**反映先（予定）**: UX-01 SC-003, SC-004 の各画面の説明。UX-02 CP-009, CP-010 のレイアウト指示。

---

## 3. 正誤の表記（合格・不合格）

| 項目 | 現行仕様書 | ワイヤーフレームでの変更 |
|------|------------|--------------------------|
| 表記 | 「正解」「不正解」（UX-01, DATA-01 の isCorrect 等） | **「合格」「不合格」**と表記する。履歴一覧（SC-005）、履歴詳細（SC-006）、採点結果の表示で統一。 |

**反映先（予定）**: UX-01 の SC-005, SC-006 の説明。UX-02 CP-005, CP-006, CP-009。必要に応じて DATA-01 の History や API レスポンスの文言方針。REQ-01 の「正解」表現と用語の整合を検討。

---

## 4. ヘッダ（共通レイアウト CP-001）

| 項目 | 現行仕様書 | ワイヤーフレームでの変更 |
|------|------------|--------------------------|
| 内容 | 「ヘッダ・パンくず・メインコンテンツ領域」とのみ記載 | **ヘッダ部分を具体的に作成**。ロゴまたはタイトル、必要に応じてナビゲーション（問題一覧・履歴・管理への導線）を配置する。 |

**反映先（予定）**: UX-02 CP-001 の「レイアウト（共通）」の説明を具体化。

---

## 5. 管理ダッシュボードの削除操作（SC-007）

| 項目 | 現行仕様書 | ワイヤーフレームでの変更 |
|------|------------|--------------------------|
| 削除 | 「新規作成・編集・削除」とあるが、削除の対象・操作が不明 | **操作列に「編集」「削除」の両方を記載**。各行（問題）ごとに「編集」「削除」のリンクまたはボタンを並べる。削除の具体的フロー（確認ダイアログ等）は未定義の場合はワイヤーフレームでは「削除」の項目を置くのみとする。 |

**反映先（予定）**: UX-01 SC-007 の「主要 UI 要素」に、問題一覧の操作列で「編集」「削除」を並べる旨を追記。必要に応じて REQ-01 FR-F07 の管理画面の操作を補足。
