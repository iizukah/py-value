# 99_history

仕様変更の経緯と、レビュー結果を記録するディレクトリです。

## いつ何を書くか
- **仕様変更**: 仕様書・設計の変更があったとき。日付・変更内容・関連ドキュメントIDを記載する。
- **レビュー結果**: レビューアが直近のコミットを対象にレビューした結果。何を確認し、問題があれば指摘内容を記載する。

## 記録粒度
- 後から読んでも**コンテキストが分かる**粒度で書く。「修正しました」ではなく、どの仕様IDのどの部分を、なぜ変えたか（または何を確認したか）を明示する。

## ファイル命名
- リポジトリ全体で**1本の連番**。`0001_概要.md`, `0002_API変更.md` のように、4桁ゼロパディング＋連番＋アンダースコア＋簡潔な説明 とする。
